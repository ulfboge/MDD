# Museum matching review data

Generated review files for MDD type-specimen museum prefix matching and estimated type-locality geocoding.

**Layout:**

| Folder | Purpose |
|--------|---------|
| [`geocoding/`](geocoding/) | Active estimated type-locality CSV + Nominatim cache |
| [`museum/`](museum/) | Current museum-matching audit outputs (regenerated by scripts) |
| [`qc/`](qc/) | New QC export batches (completed batches in [`archive/qc/`](archive/qc/)) |
| [`archive/`](archive/README.md) | Closed research notes and finished QC batches |

## Matching rule (web app)

`UPPER(type_voucher) LIKE UPPER(abbreviation) || '%'`, longest abbreviation wins.

Example: `NHRM` required its own metadata row; `MN` matches `MN 31910` but not `MNCN` / `MNHN` vouchers.
